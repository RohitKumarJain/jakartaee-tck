Setup and configuration of Jakarta EE 8 CI and CTS 8 for testing a standalone
Jakarta Messaging Resource Adapter

  • Install Java SE 8, Jakarta EE 8 CI, and CTS 8
  • Configure and edit the CTS 8 ts.jte file
    • Configure Jakarta EE 8 CI for CTS 8 and the standalone Jakarta Messaging Resource Adapter. These instructions are for the Jakarta EE 8 CI, Eclipse GlassFish 5.1
    • Modify the Jakarta Messaging CTS MDB and Resource Adapter tests runtime deployment descriptors
  • Run the Jakarta Messaging CTS tests using command line ant targets
  • Restore the Jakarta Messaging CTS MDB and Resource Adapter tests runtime deployment descriptors
  • Reconfigure Jakarta EE 8 CI for CTS 8 after testing standalone Jakarta Messaging Resource Adapter

Install Java SE 8, Jakarta EE 8 CI, and CTS 8

Before you can run the Jakarta Messaging CTS tests against a standalone Jakarta Messaging Resource Adapter using the Jakarta EE 8 CI you must install the following components

  • Java SE 8
  • Java EE 8 CI, for example, Eclipse GlassFish 5.1
  • Jakarta EE 8 Platform CTS 8 (CTS 8)

Complete the following steps to set up your environment:

  1. set the following environment variables in your shell environment:
    1. JAVA_HOME to the directory where the Java SE 8 software has been installed
    2. JAVAEE_HOME to the directory where the Jakarta EE 8 CI software has been installed
    3. TS_HOME to the directory where the CTS 8 software has been installed
    4. ANT_HOME to the directory where ant resides in your test environment

    For example, you would set the following if running on Solaris or Linux:

    % setenv JAVA_HOME /java8
% setenv JAVAEE_HOME /glassfish6/glassfish
% setenv TS_HOME /jakartaeetck
% setenv ANT_HOME /anthome

    Or, if you are running on Windows you would set the following:

    c:\> set JAVA_HOME=C:\java
c:\> set JAVAEE_HOME=C:\glassfish6\glassfish
c:\> set TS_HOME=C:\jakartaeetck
c:\> set ANT_HOME=C:\apacheant

    Update your PATH environment variable to include the following paths (Unix Example):

    % setenv PATH $TS_HOME/bin:$JAVA_HOME/bin:$ANT_HOME/bin:$JAVAEE_HOME/bin:$PATH

    For Windows update your PATH environment as follows:

    c:\ set PATH=%TS_HOME%\bin;%JAVA_HOME%\bin;%ANT_HOME%\bin;%JAVAEE_HOME%\bin;%PATH%

    2. Configure and edit the CTS 8 ts.jte file

    At the bottom of the ts.jte file we added properties and documentation for testing a standalone Jakarta Messaging Resource Adapter using the Jakarta EE 8 CI, Eclipse GlassFish 5.1. Please read the Jakarta Messaging Resource Adapter documentation at the bottom of the ts.jte before proceeding. It will help in understanding this section of the CTS 8 Users Guide.

    1. Set the following in your ts.jte file

      1. jakartaee.home set this property to location where Jakarta EE 8 CI is installed
      2. orb.host use default value or enter a new host name
      3. orb.port use default value or enter a new port number
      4. test.sa.jmsra set this property to true
      5. jmsra.rarfile set to the location of the standalone Jakarta Messaging Resource Adapter RAR file
      6. jmsra.jarfile set to the location of the standalone Jakarta Messaging Resource Adapter JAR file
      7. jmsra.name set this property to the name that refers to the Jakarta Messaging Resource Adapter under test

      Verify rest of ts.jte settings by consulting the CTS 8 Users Guide.

      Example settings if running on Solaris or Linux:

      jakartaee.home=/glassfish6/glassfish
orb.host=localhost
orb.port=3699
test.sa.jmsra=true
jmsra.rarfile=/files/myjmsra/myjmsra.rar
jmsra.jarfile=/files/myjmsra/myjmsra.jar
jmsra.name=myjmsra

      Example settings if running on Windows:

      jakartaee.home=c:/glassfish6/glassfish
orb.host=localhost
orb.port=3699
test.sa.jmsra=true
jmsra.rarfile=c:/files/myjmsra/myjmsra.rar
jmsra.jarfile=c:/files/myjmsra/myjmsra.jar
jmsra.name=myjmsra

      The jmsra.jarfile which contains all the Jakarta Messaging Resource Adapter classes needs to be added to the AppClient classpath in the ts.jte file. This JAR file also gets copied to the appropriate directory within your Jakarta EE 8 environment during config.vi which is described in the next section. For the Jakarta EE 8 CI, Eclipse GlassFish 5.1 that would be the JAVAEE_HOME/lib directory.

      Add ${jmsra.jarfile} to the beginning or at the end of AppClient classpath:

      APPCPATH=list of classes and jars followed by ${pathsep}${jmsra.jarfile} \

    Configure Jakarta EE 8 CI for CTS 8 and the standalone Jakarta Messaging Resource Adapter

    Invoke the config.vi ant task to configure the Jakarta EE 8 CI for CTS 8 and the standalone Jakarta Messaging Resource Adapter

    % cd $TS_HOME/bin
% ant config.vi

    Or, if you are running on Windows you would type:

    cd %TS_HOME%\bin
ant config.vi

    During config.vi ant scripts get executed which configures your Jakarta EE 8 environment for CTS 8. One of the ant scripts that gets executed will configure the standalone Jakarta Messaging Resource Adapter, deploy the standalone Jakarta Messaging Resource Adapter, copy the JAR file containing the classes for the standalone Jakarta Messaging Resource Adapter to JAVAEE_HOME/lib, create the Jakarta Messaging connector connection pools, create the Jakarta Messaging connector resources, and finally create the Jakarta Messaging administration objects needed for CTS 8. The ant scripts that get called are the following:

    1. TS_HOME/bin/xml/impl/glassfish/jmsra.xml
    2. TS_HOME/bin/xml/impl/glassfish/templates/create.jmsra.template

    These scripts are written for the standalone Generic Jakarta Messaging Resource Adapter (GenericJMSRA) for the Jakarta EE 8 CI. If using another Jakarta EE 8 Platform Environment then you will need to rewrite these scripts for that Jakarta EE 8 Platform Environment. If using another standalone Jakarta Messaging Resource Adapter then these scripts will need to be rewritten for that Jakarta Messaging Resource Adapter.

    The script TS_HOME/bin/xml/impl/glassfish/jmsra.xml calls the template file TS_HOME/bin/xml/impl/glassfish/templates/create.jmsra.template which handles all the creation of the Jakarta Messaging connector connection pools, the Jakarta Messaging connector resources and the Jakarta Messaging administration objects.

    After config.vi completes your Jakarta EE 8 environment is almost ready for running the CTS 8 Jakarta Messaging tests.

    Modify the Jakarta Messaging CTS MDB and Resource Adapter tests runtime deployment descriptors

    Now that we have deployed and configured the standalone Jakarta Messaging Resource Adapter and created all the Jakarta Messaging connector connection pools, Jakarta Messaging connector resources, and Jakarta Messaging administration objects we now have to modify all the sun-ejb-jar runtime deployment descriptor xml files for all the Jakarta Messaging CTS MDB and Resource Adapter tests. We have an ant task that handles this.

    Invoke the following ant task to modify all the Jakarta Messaging CTS MDB and Resource Adapter sun-ejb-jar runtime deployment descriptor xml files by adding the necessary <mdb-resource-adapter> configuration info.

    % cd $TS_HOME/bin
% ant -f xml/impl/glassfish/jmsra.xml modify-jmsmdbejbxml

    Or, if you are running on Windows you would type:

    c:\> cd %TS_HOME%\bin
c:\> ant -f xml\impl\glassfish\jmsra.xml modify-jmsmdbejbxml

    This ant target modifies all the sun-ejb-jar runtime deployment descriptor xml files in the distribution directory of the Jakarta Messaging CTS MDB and Resource Adapter test directories that exist under TS_HOME/src/com/sun/ts/tests/jms/ee/mdb and TS_HOME/src/com/sun/ts/tests/jms/ee20/ra.

    So it modifies all the sun-ejb-jar runtime deployment descriptor xml files for all subdirectories under TS_HOME/dist/com/sun/ts/tests/jms/ee/mdb and TS_HOME/dist/com/sun/ts/tests/jms/ee20/ra.

    The modified sun-ejb-jar runtime deployment descriptor xml files exist under the directory: TS_HOME/src/com/sun/ts/tests/jms/commonee/xml/descriptors/genericra. They are simply copied into the correct distribution test directory under TS_HOME/dist/com/sun/ts/tests/jms/ee/mdb and TS_HOME/dist/com/sun/ts/tests/jms/ee20/ra.

    What is added to the sun-ejb-jar runtime deployment descriptor xml files is the <mdb-resource-adapter> info for the standalone Jakarta Messaging Resource Adapter being tested. In our case the Generic Jakarta Messaging Resource Adapter (GenericJMSRA).

    If using another Jakarta EE 8 Platform Environment then these runtime deployment descriptor xml files will be vendor specific. In this case you will need to modify the ant script to handle the vendor specific runtime deployment descriptor xml files.

    Run the Jakarta Messaging CTS tests using command line ant targets

    To run the Jakarta Messaging CTS tests type the following:

    1. cd $TS_HOME/src/com/sun/ts/tests/jms
    2. ant runclient

    Or, if you are running on Windows you would type:

    1. cd %TS_HOME%\src\com\sun\ts\tests\jms
    2. ant runclient

    Restore the Jakarta Messaging CTS MDB and Resource Adapter tests runtime deployment descriptors

    After you run the Jakarta Messaging CTS tests against your standalone Jakarta Messaging Resource Adapter you must remember to restore the Jakarta Messaging CTS MDB and Resource Adapter tests. We have an ant task that handles this.

    Invoke the following ant task to restore all the Jakarta Messaging CTS MDB and Resource Adapter sun-ejb-jar runtime deployment descriptor xml files. This restores the sun-ejb-jar runtime deployment descriptor xml file back to there previous state.

    % cd $TS_HOME/bin
% ant -f xml/impl/glassfish/jmsra.xml restore-jmsmdbejbxml

    Or, if you are running on Windows you would type:

    c:\> cd %TS_HOME%\bin
c:\> ant -f xml\impl\glassfish\jmsra.xml restore-jmsmdbejbxml

    If using another Jakarta EE 8 Platform Environment then these runtime deployment descriptor xml files will be vendor specific. In this case you will need to modify the ant script to handle the vendor specific runtime deployment descriptor xml files.

    Reconfigure Jakarta EE 8 CI for CTS 8 after testing standalone Jakarta Messaging Resource Adapter

    Invoke the clean.vi ant task to unconfigure the Jakarta EE 8 CI for CTS 8 after testing the standalone Jakarta Messaging Resource Adapter

    % cd $TS_HOME/bin
% ant clean.vi

    Or, if you are running on Windows you would type:

    cd %TS_HOME%\bin
ant clean.vi

    Now we will reconfigure the Jakarta EE 8 CI for CTS 8 testing but first we must edit the ts.jte file and reset the Jakarta Messaging Resource Adapter properties back to orginal state.

    1. Set the following in your ts.jte file

      1. jakartaee.home set this property to location where Jakarta EE 8 CI is installed
      2. orb.host use default value or enter a new host name
      3. orb.port use default value or enter a new port number
      4. test.sa.jmsra set this property back to false
      5. jmsra.rarfile unset this property
      6. jmsra.jarfile unset this property
      7. jmsra.name restore this property back to jmsra to refer to the Jakarta Messaging Resource Adapter for the Jakarta EE 8 CI

      Verify the rest of your ts.jte settings by consulting the CTS 8 Users Guide.

      Example settings if running on Solaris or Linux:

      jakartaee.home=/glassfish6/glassfish
orb.host=localhost
orb.port=3699
test.sa.jmsra=false
jmsra.rarfile=
jmsra.jarfile=
jmsra.name=jmsra

      Example settings if running on Windows:

      jakartaee.home=c:/glassfish6/glassfish
orb.host=localhost
orb.port=3699
test.sa.jmsra=false
jmsra.rarfile=
jmsra.jarfile=
jmsra.name=jmsra

      Now invoke the config.vi ant task to reconfigure the Java EE 8 CI for CTS 8

      % cd $TS_HOME/bin
% ant config.vi

      Or, if you are running on Windows you would type:

      cd %TS_HOME%\bin
ant config.vi
